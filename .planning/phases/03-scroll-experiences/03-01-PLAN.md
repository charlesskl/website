**Create `assets/animation/scroll-reveals.js`** — Scroll-triggered clip-path geometric wipe reveals for images.

IIFE pattern with 'use strict' (matching Phase 1 convention). Register via `window.RR.register('scrollReveals', {init, kill, refresh})`.

The module must:
1. **CDN graceful degradation**: Check `typeof gsap !== 'undefined'` and `typeof ScrollTrigger !== 'undefined'`. If missing, return gracefully.
2. **Reduced motion gate**: If `window.RR.state.hasReducedMotion` is true, `init()` is a no-op — images remain fully visible.
3. **Coarse pointer gate**: If `window.RR.breakpoints && window.RR.breakpoints.isCoarsePointer`, use simplified reveals (just opacity fade, no clip-path) — complex clip-path animations cause jank on mobile GPUs.
4. **Target selection**: Query elements that should receive clip-path reveals:
   - `.photo-panel` elements (capability pages — each contains an `<img>`)
   - `.cap-card-img` elements (index pages — product image inside capability cards)
   - `#historyBg` (index/about pages — full-bleed history background)
5. **Clip-path wipe reveal implementation**:
   a) For `.photo-panel` elements — **top-to-bottom wipe**:
      - Set initial: `gsap.set(panel, { clipPath: 'inset(0 0 100% 0)' })` — fully clipped from bottom
      - Animate to: `clipPath: 'inset(0 0 0% 0)'` — reveals top to bottom
      - Use `scrollTrigger: { trigger: panel, start: 'top 85%', once: true }`
      - Duration: 1.0s, ease: 'power3.inOut'
      - Stagger panels in the same `.photo-panels` container: 0.15s between each
   b) For `.cap-card-img` elements — **diagonal wipe**:
      - Set initial: `gsap.set(img, { clipPath: 'polygon(0 0, 0 0, 0 100%, 0 100%)' })` — clipped from left
      - Animate to: `clipPath: 'polygon(0 0, 100% 0, 100% 100%, 0 100%)'` — reveals left to right
      - Use `scrollTrigger: { trigger: cardParent, start: 'top 80%', once: true }`
      - Duration: 0.8s, ease: 'power2.out'
   c) For `#historyBg` — **diagonal wipe from top-left**:
      - Set initial: `gsap.set(bg, { clipPath: 'polygon(0 0, 0 0, 0 0, 0 0)' })` — invisible
      - Animate to: `clipPath: 'polygon(0 0, 100% 0, 100% 100%, 0 100%)'` — full reveal
      - Use `scrollTrigger: { trigger: '#historySection', start: 'top 60%', once: true }`
      - Duration: 1.2s, ease: 'power3.inOut'
6. **Track all ScrollTrigger instances and timelines** in internal arrays for cleanup.
7. **kill()**: Kill all tracked timelines and ScrollTrigger instances. Reset clip-path on all affected elements using `gsap.set(el, { clearProps: 'clipPath' })`.
8. **refresh()**: Kill all, then re-init.

**Create `assets/animation/scroll-skew.js`** — Velocity-based skew effect that makes elements subtly lean during fast scroll.

IIFE pattern with 'use strict'. Register via `window.RR.register('scrollSkew', {init, kill, refresh})`.

The module must:
1. **CDN graceful degradation**: Check `typeof gsap !== 'undefined'`. If missing, return gracefully.
2. **Reduced motion gate**: No-op if `window.RR.state.hasReducedMotion`.
3. **Coarse pointer gate**: Skip on touch devices — skew feels wrong on touch scroll with inertia.
4. **Lenis velocity integration**:
   - Use `window.RR.lenis.on('scroll', callback)` to read `e.velocity` on each scroll event
   - If Lenis is not available (`!window.RR.lenis`), fall back to no-op gracefully
5. **Skew algorithm**:
   - Create a `gsap.quickSetter` for each target element's `skewY` transform
   - On each Lenis scroll event, calculate `skewAmount = Math.min(Math.max(velocity * 0.3, -4), 4)` — clamp to max 4 degrees
   - Use `gsap.to()` on a proxy object to smooth the skew transition (don't apply raw velocity directly — it will jitter). Tween proxy.skew toward the target skew with `duration: 0.3, ease: 'power2.out'`, and use `onUpdate` to apply via quickSetter.
   - When velocity drops below threshold (< 0.1), tween skew back to 0 with `duration: 0.6, ease: 'elastic.out(1, 0.5)'` for a satisfying return bounce.
6. **Target selection**: Apply skew to sections that benefit from the effect:
   - `.clip-section` elements (intro-statement, caps-section, history-section on index)
   - `.editorial` elements (capability pages)
   - `.cta-section` elements (capability and contact pages)
   - Exclude `.hero`, `.page-hero`, `.sub-hero`, `.about-hero` — hero sections should not skew
   - Exclude `footer` — footer reveal should not skew
7. **Performance**: Use `will-change: transform` on targets only while skew is active. Remove when skew returns to 0.
8. **Track the Lenis scroll listener reference** for cleanup.
9. **kill()**: Remove Lenis scroll listener, kill any running skew tweens, clear skew transforms on all targets, remove will-change.
10. **refresh()**: Kill all, re-init.
