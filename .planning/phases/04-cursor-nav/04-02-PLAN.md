**Create `assets/animation/navbar.js`** — Smart scroll-direction navbar hide/show replacing initNavbar() in shared.js.

The module must:
1. **CDN graceful degradation**: If GSAP is missing, fall back to CSS transitions (add/remove a class). The hide/show behavior should work either way.
2. **Reduced motion gate** (NAV-01): If reduced motion, still hide/show but use instant state change (no animated transition). Use `gsap.set()` instead of `gsap.to()`.
3. **Target**: `#navbar` element. Early return if not found.
4. **Scroll direction detection using Lenis** (NAV-01):
   - If `window.RR.lenis` available, use `window.RR.lenis.on('scroll', callback)` to read scroll direction and position.
   - If Lenis not available, fall back to `window.addEventListener('scroll', callback, {passive: true})` with manual direction detection.
   - Track last scroll position for direction detection.
5. **Smart hide/show behavior** (NAV-01):
   - **Scrolling DOWN** past threshold (e.g., 80px from top): Hide navbar with `gsap.to(navbar, {yPercent: -100, duration: 0.4, ease: 'power3.inOut'})`.
   - **Scrolling UP**: Show navbar with `gsap.to(navbar, {yPercent: 0, duration: 0.35, ease: 'power3.out'})`.
   - **Near top of page** (scrollY < 80): Always show navbar, clear any transforms.
   - **Debounce**: Don't trigger on every scroll event. Use a scroll distance threshold of 10px to prevent jitter on small scroll movements.
   - **Don't hide during mobile menu open**: Check if mobile menu is open (`.nav-links.open` or check `window.RR.state.menuOpen`). If menu is open, don't hide.
6. **Scrolled state styling**: Preserve the existing `.scrolled` class behavior — add class when scrollY > 60, remove when < 60. This adds the frosted glass background.
   ```javascript
   navbar.classList.toggle('scrolled', scrollY > 60);
   ```
7. **Prevent hide during hero entrance**: On page load, don't hide the navbar for the first 2.5 seconds (hero timeline is playing). Use a flag `var canHide = false` and `setTimeout(function() { canHide = true; }, 2500)`.
8. **State tracking**: Set `window.RR.state.navHidden = true/false` so other modules (like mobile menu) can know the navbar state.
9. **Track Lenis scroll listener** for cleanup.
10. **kill()**: Remove Lenis scroll listener (or window scroll listener), clear navbar transforms with `gsap.set(navbar, {clearProps: 'transform'})`, remove `.scrolled` class.
11. **refresh()**: Kill then re-init.

IMPORTANT: Do NOT modify shared.js. The existing initNavbar() in shared.js will still run and add/remove the `.scrolled` class. navbar.js ALSO adds/removes `.scrolled`. This is fine — both check `scrollY > 60` and the result is the same. The key addition is the yPercent hide/show which the old code explicitly did NOT do.

**Create `assets/animation/page-transition.js`** — Enhanced clip-path overlay page transition replacing initPageTransitions() in shared.js.

IIFE pattern with 'use strict'. Register via `window.RR.register('pageTransition', {init, kill, refresh})`.

The module must:
1. **CDN graceful degradation**: If GSAP missing, navigate immediately without transition. The page must always navigate — never trap the user on a broken transition.
2. **Reduced motion gate**: If reduced motion, navigate immediately without animation.
3. **DOM creation** — Create the transition overlay:
   ```javascript
   var overlay = document.createElement('div');
   overlay.className = 'rr-page-transition';
   document.body.appendChild(overlay);
   ```
   Use `rr-` prefix to avoid collision with existing `.page-transition` element created by shared.js.
4. **Exit transition** (NAV-03) — On internal link click:
   a) Detect internal links: `href` doesn't start with `#`, `mailto:`, `tel:`, `http`, `javascript:`, and link has no `target` attribute.
   b) Prevent default navigation.
   c) Set `window.RR.state.isTransitioning = true` — this flag tells other modules (cursor, navbar, etc.) that a transition is in progress.
   d) Build exit timeline:
      ```javascript
      var exitTl = gsap.timeline({
        onComplete: function() {
          window.location.href = href;
        }
      });
      exitTl.to(overlay, {
        clipPath: 'inset(0 0% 0 0)',  // green wipe covers from left
        duration: 0.45,
        ease: 'power3.inOut'
      });
      ```
   e) Store transition state in sessionStorage: `sessionStorage.setItem('rr_page_wipe', '1')`.
5. **Enter transition** — On page load, check for transition state:
   ```javascript
   if (sessionStorage.getItem('rr_page_wipe') === '1') {
     sessionStorage.removeItem('rr_page_wipe');
     // Set overlay to fully covering
     gsap.set(overlay, { clipPath: 'inset(0 0 0 0)' });
     // Reveal by wiping overlay off to the right
     gsap.to(overlay, {
       clipPath: 'inset(0 0 0 100%)',
       duration: 0.5,
       ease: 'power3.inOut',
       delay: 0.05,
       onComplete: function() {
         gsap.set(overlay, { clipPath: 'inset(0 100% 0 0)' }); // reset
       }
     });
   }
   ```
6. **Prevent double-click**: After first link click triggers transition, ignore subsequent clicks. Use the `isTransitioning` flag.
7. **Timeout safety**: If navigation hasn't happened within 3 seconds, force navigate. This prevents the user being stuck on a green screen.
   ```javascript
   setTimeout(function() {
     if (window.RR.state.isTransitioning) {
       window.location.href = href;
     }
   }, 3000);
   ```
8. **Override old transition**: The old initPageTransitions() in shared.js creates `.page-transition` and uses `sessionStorage.getItem('page_wipe')`. page-transition.js creates `.rr-page-transition` and uses `sessionStorage.getItem('rr_page_wipe')`. Different keys and elements — no collision. To prevent BOTH transitions from firing on the same click, page-transition.js should use `e.stopPropagation()` on the click handler (or use capture phase) to prevent the old handler from also intercepting.
   Better approach: page-transition.js uses capture phase listener:
   ```javascript
   document.addEventListener('click', handleClick, true); // capture phase — runs before shared.js bubble handler
   ```
   In handleClick, after calling `e.preventDefault()`, also call `e.stopPropagation()` to prevent the old handler from firing.
9. **CSS**: Add to shared.css:

   ```css
   }
   ```
10. **kill()**: Remove overlay element, remove click listener, clear sessionStorage key.
11. **refresh()**: No-op (transition state shouldn't reset on breakpoint change).
