**Create `assets/animation/cursor.js`** — Four-state custom cursor system replacing the existing initCursor() in shared.js.

IIFE pattern with 'use strict' (matching Phase 1 convention). Register via `window.RR.register('cursor', {init, kill, refresh})`.

The module must:
1. **Touch device gate** (CURSOR-03): Use BOTH checks:
   - `window.RR.breakpoints && window.RR.breakpoints.isCoarsePointer` (from responsive.js matchMedia)
   - Fallback if breakpoints not yet loaded: `window.matchMedia('(pointer: coarse)').matches || window.matchMedia('(hover: none)').matches`
   - If touch detected, `init()` must be a COMPLETE no-op — no DOM elements created, no event listeners attached, nothing. The cursor system must be ABSENT, not hidden.
2. **CDN graceful degradation**: Check `typeof gsap !== 'undefined'`. If GSAP missing, still create the cursor but use CSS transitions instead of GSAP for state changes. The cursor should work without GSAP (core follows mouse via RAF), only enhanced animations need GSAP.
3. **Reduced motion gate**: If `window.RR.state.hasReducedMotion`, cursor still appears (it's a UI element) but state transitions are instant (no animated size/opacity changes).
4. **DOM creation** — Create 3 elements on init():
   ```javascript
   var dot = document.createElement('div');
   dot.className = 'rr-cursor-dot';
   document.body.appendChild(dot);

   var ring = document.createElement('div');
   ring.className = 'rr-cursor-ring';
   document.body.appendChild(ring);

   var label = document.createElement('div');
   label.className = 'rr-cursor-label';
   document.body.appendChild(label);
   ```
   Use `rr-` prefix to avoid collision with existing `.cursor-dot` etc. in shared.css (which is still used by the old initCursor in shared.js until it's disabled).
5. **Mouse tracking**:
   - Dot: follows mouse position INSTANTLY using direct `style.transform = 'translate3d(Xpx, Ypx, 0)'` in mousemove handler (no GSAP overhead on the hot path).
   - Ring: follows with smooth lag. If GSAP available, use `gsap.quickTo(ring, 'x', {duration: 0.5, ease: 'power3'})` and `gsap.quickTo(ring, 'y', {duration: 0.5, ease: 'power3'})`. If no GSAP, use manual lerp in RAF loop (factor 0.15).
   - Label: follows ring position, centered with translate(-50%, -50%).
   - Hide dot + ring (opacity: 0) until first mousemove event to prevent flash at 0,0.
6. **Four cursor states** (CURSOR-01):
   a) **Default (dot)**: Small 6px dot + 36px ring border. The resting state.
   b) **Link (expand)**: Ring expands to 56px with slight background tint. Triggered on `a, button, .magnetic, [role="button"]` hover. Dot shrinks to 3px.
   c) **CTA (scale + label)**: Ring expands to 80px, shows label text. Triggered on elements with `[data-cursor]` attribute — `label.textContent = el.getAttribute('data-cursor')`. Also triggered on `.cap-card` (shows "VIEW"), `.photo-panel img` (shows "VIEW").
   d) **Crosshair**: Ring shrinks to 24px with crosshair lines. Triggered on `img:not(.nav-logo img):not(.loader-logo img), .img-reveal`. Dot becomes crosshair (via CSS class).
7. **State detection**: Use event delegation with `mouseover` and `mouseout` on `document`:
   ```javascript
   document.addEventListener('mouseover', function(e) {
     var target = e.target;
     // Check in priority order: CTA > link > image > default
     var cta = target.closest('[data-cursor], .cap-card, .photo-panel');
     var link = target.closest('a, button, .magnetic, [role="button"]');
     var img = target.closest('img, .img-reveal');

     if (cta) { setState('cta', cta); }
     else if (link) { setState('link'); }
     else if (img) { setState('crosshair'); }
   });
   ```
   On mouseout, clear state back to default if leaving the triggering element.
8. **State application**: Use GSAP for smooth state transitions if available:
   ```javascript
   function setState(state, el) {
     // Remove all state classes
     ring.className = 'rr-cursor-ring';
     dot.className = 'rr-cursor-dot';
     label.className = 'rr-cursor-label';

     if (state === 'link') {
       ring.classList.add('is-link');
       dot.classList.add('is-small');
     } else if (state === 'cta') {
       ring.classList.add('is-cta');
       dot.classList.add('is-hidden');
       var text = el.getAttribute('data-cursor') || 'VIEW';
       label.textContent = text;
       label.classList.add('is-visible');
     } else if (state === 'crosshair') {
       ring.classList.add('is-crosshair');
       dot.classList.add('is-crosshair');
     }
   }
   ```
9. **Hide native cursor**: Add `cursor: none` to `body` when cursor module is active. Restore on kill().
10. **Track all event listeners and elements** for cleanup.
11. **kill()**: Remove all 3 DOM elements, remove all event listeners, restore `body` cursor, cancel any RAF.
12. **refresh()**: Kill then re-init.

IMPORTANT: Do NOT modify shared.js. The old initCursor() and initMagnetic() in shared.js will still run, but the new cursor.js creates DIFFERENT elements with `rr-` prefix. To prevent the old cursor from showing alongside the new one, add a CSS rule: `.cursor-dot, .cursor-ring, .cursor-label { display: none !important; }` at the end of the new cursor CSS block (this overrides the old system). This is safer than modifying shared.js which could break other modules.

**Create `assets/animation/magnetic.js`** — Magnetic button attraction with GSAP spring physics replacing initMagnetic() in shared.js.

IIFE pattern with 'use strict'. Register via `window.RR.register('magnetic', {init, kill, refresh})`.

The module must:
1. **Touch device gate** (CURSOR-03): Same as cursor.js — complete no-op on touch devices.
2. **CDN graceful degradation**: Requires GSAP for spring physics. If `typeof gsap === 'undefined'`, return gracefully — magnetic effect is purely enhancement.
3. **Reduced motion gate**: If `window.RR.state.hasReducedMotion`, skip — buttons work normally without magnetic pull.
4. **Target selection** (CURSOR-02): Query all magnetic elements:
   ```javascript
   var targets = document.querySelectorAll('[data-magnetic], .btn, .nav-cta, .magnetic');
   ```
   The `[data-magnetic]` attribute allows per-element opt-in with custom strength.
5. **Magnetic attraction implementation** using `gsap.quickTo()`:
   - For each target element, create quickTo tweens for x and y:
     ```javascript
     var xTo = gsap.quickTo(el, 'x', { duration: 0.8, ease: 'elastic.out(1, 0.3)' });
     var yTo = gsap.quickTo(el, 'y', { duration: 0.8, ease: 'elastic.out(1, 0.3)' });
     ```
   - On `mousemove` within the element:
     ```javascript
     var rect = el.getBoundingClientRect();
     var strength = parseFloat(el.dataset.magnetic) || 0.35;
     var relX = e.clientX - rect.left - rect.width / 2;
     var relY = e.clientY - rect.top - rect.height / 2;
     xTo(relX * strength);
     yTo(relY * strength);
     ```
   - On `mouseleave`:
     ```javascript
     xTo(0);
     yTo(0);
     ```
   - The `quickTo` API is more performant than `gsap.to()` for rapid updates because it reuses the tween internally.
6. **Attraction radius** (CURSOR-02 — "attract as it approaches"):
   - Add a `mouseenter` detection zone using a transparent pseudo-element or by monitoring mousemove on the parent container.
   - Alternative simpler approach: on `mousemove` on the element's parent (or document), check distance to element center. If within 80px radius, apply a subtle pull (strength * 0.15). This creates the "approaching attraction" effect.
   - Implementation: For each magnetic element, listen on `mousemove` at a broader scope. Calculate distance from mouse to element center. If distance < attraction radius (80px), apply proportional pull. If distance > radius, reset.
   ```javascript
   el.parentElement.addEventListener('mousemove', function(e) {
     var rect = el.getBoundingClientRect();
     var cx = rect.left + rect.width / 2;
     var cy = rect.top + rect.height / 2;
     var dx = e.clientX - cx;
     var dy = e.clientY - cy;
     var dist = Math.sqrt(dx * dx + dy * dy);
     var radius = parseFloat(el.dataset.magneticRadius) || 80;

     if (dist < radius) {
       var pull = 1 - (dist / radius); // 0 at edge, 1 at center
       xTo(dx * strength * pull * 0.5);
       yTo(dy * strength * pull * 0.5);
     }
   });
   ```
   When the mouse is directly over the element, the existing mousemove handler takes over with full strength.
7. **Track all event listeners** for cleanup (store references in an array).
8. **kill()**: Remove all event listeners, reset all magnetic elements to x:0, y:0 using `gsap.set()`.
9. **refresh()**: Kill then re-init (handles dynamic content).
