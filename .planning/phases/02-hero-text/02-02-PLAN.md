**Create `assets/animation/scroll-text.js`** — Scroll-triggered masked word/line reveal module.

IIFE pattern with 'use strict' (matching Phase 1 convention). Register via `window.RR.register('scrollText', {init, kill, refresh})`.

The module must:
1. **CDN graceful degradation**: Check `typeof gsap !== 'undefined'` and `typeof ScrollTrigger !== 'undefined'` and `typeof SplitText !== 'undefined'`. If any missing, return gracefully.
2. **Reduced motion gate**: If `window.RR.state.hasReducedMotion` is true, `init()` is a no-op — headings remain visible as-is.
3. **Target selection**: On `init()`, query ALL section headings that should animate:
   - `h2.section-title` (index, careers pages — may have `.reveal-up` class)
   - `h2.history-heading` (index, about pages)
   - `h2.reveal-up` that are NOT hero titles (contact, careers speculative section)
   - `.section-label.reveal-up` (small label text above headings)
   - Exclude any element inside `.hero-content`, `.about-hero-content`, `.sub-hero-content` (hero elements are handled by hero.js)
4. **For each heading element**, create a scroll-triggered masked reveal:
   a) Use `window.RR.textSystem.split(element, { type: 'words,lines', mask: 'overflow' })` — split into words and lines. The `mask: 'overflow'` option creates overflow:hidden wrappers automatically.
   b) After the split Promise resolves, set initial state: `gsap.set(words, { yPercent: 100 })` — words start hidden below their mask.
   c) Create a `gsap.timeline()` with `scrollTrigger: { trigger: element, start: 'top 85%', once: true }`.
   d) Animate words: `gsap.to(words, { yPercent: 0, duration: 0.7, ease: 'power3.out', stagger: 0.04 })`.
   e) For `.section-label` elements, use a simpler reveal: `{ yPercent: 100, duration: 0.5, stagger: 0.03 }` with slightly earlier trigger `start: 'top 88%'` so the label animates in before the heading below it.
5. **Track all created ScrollTrigger instances and timelines** in an internal array for cleanup.
6. **kill()**: Kill all tracked timelines and ScrollTrigger instances. Call `window.RR.textSystem.killAll()` is NOT needed here — text-system tracks its own splits.
7. **refresh()**: Kill all, then re-init (needed when matchMedia reverts).
8. **Init timing**: Wrap the entire init logic inside `window.RR.onFontsReady()` to ensure SplitText calculates line breaks correctly after custom fonts load.

IMPORTANT constraints:
- Do NOT apply scrollTrigger to individual tweens inside a timeline (Pitfall 3). Apply it to the timeline itself.
- Use `gsap.fromTo()` pattern if needed, but since we're setting initial state with `gsap.set()` then animating with `gsap.to()`, this is fine.
- The `.reveal-up` class currently has inline CSS `opacity: 0; transform: translateY(24px)` on some elements. The SplitText split will override the element's content structure. After splitting, the parent element needs `opacity: 1; transform: none` so the words inside (which are masked) are the ones controlling visibility. Add a `gsap.set(element, { opacity: 1, y: 0, clearProps: 'transform' })` before setting word initial states.
- Do NOT remove the `reveal-up` class from HTML — Phase 3 might use it as a selector. Just override its visual effect by setting opacity/transform on the element directly.

**Create `assets/animation/text-scramble.js`** — Matrix-style text scramble effect module.

IIFE pattern with 'use strict'. Register via `window.RR.register('textScramble', {init, kill, refresh})`.

The module must:
1. **CDN graceful degradation**: Check `typeof gsap !== 'undefined'` and `typeof ScrollTrigger !== 'undefined'`. SplitText is NOT needed for this module (pure character randomization, not SplitText splitting).
2. **Reduced motion gate**: If `window.RR.state.hasReducedMotion`, no-op.
3. **Scramble algorithm**: Implement a `scrambleText(element, finalText, options)` function:
   - `chars`: String of characters to randomize with. Default: `'0123456789ABCDEF!@#$%'` (Matrix-style characters).
   - `duration`: Total scramble duration in seconds. Default: `1.2`.
   - `revealDelay`: Stagger delay before each character resolves. Default: `0.08` per character.
   - Algorithm: Use `gsap.to()` with an `onUpdate` callback. Create a proxy object `{ progress: 0 }` and tween `progress` from 0 to 1. On each update, for each character position: if `progress > (charIndex / totalChars)`, show the real character; otherwise, show a random character from the `chars` set. Set `element.textContent` on each update.
   - Use `font-variant-numeric: tabular-nums` on the element during scramble (via inline style) to prevent layout shift from varying character widths. Remove after animation completes.
4. **Target elements**: On `init()`, find elements to scramble:
   - `#stmtAccent` — the "1987" decorative number in intro-statement sections (appears on index and about pages across all languages). Trigger: ScrollTrigger on `#introStatement` section, `start: 'top 75%'`, `once: true`.
   - `#historyWatermark` — the "1987" background watermark in history sections (appears on index and about pages). Trigger: ScrollTrigger on the history section parent (`#historySection` or parent of `#historyWatermark`), `start: 'top 75%'`, `once: true`.
   - For `#stmtAccent`: the existing `statementEntrance()` in each page's inline script currently animates this element's opacity from 0 to 0.06. The scramble should REPLACE this — so text-scramble.js should set `stmtAccent` to `opacity: 0` initially, and when triggered, first set `opacity: 0.06` then scramble the text. Or better: set opacity to 0.06 and scramble simultaneously. The scramble should reveal the text character-by-character while the element fades in.
   - For `#historyWatermark`: the watermark is already visible at opacity ~0.03 (CSS). The scramble should play on first scroll into view. After the inline `historyTimeline()` function changes the watermark text via `goTo()`, the scramble should NOT replay (it's a one-time entrance effect). Set a flag `data-scrambled="true"` after initial scramble.
5. **Integration with existing inline animations**: The existing `statementEntrance()` function in index.html (and about.html) sets `gsap.set(accent, { opacity: 0 })` and then `gsap.to(accent, { opacity: 0.06 })`. text-scramble.js should handle the `stmtAccent` element INSTEAD. To avoid conflict: in Task 2, we will remove the stmtAccent-related lines from `statementEntrance()` inline code so text-scramble.js owns that element entirely.
6. **Track all ScrollTrigger instances** for cleanup.
7. **kill()**: Kill tracked ScrollTriggers, clear any running scramble tweens.
8. **refresh()**: Kill all, re-init.
9. Wrap init in `window.RR.onFontsReady()` for consistent timing (though less critical for numbers).
