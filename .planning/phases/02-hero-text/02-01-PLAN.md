**Create `assets/animation/text-system.js`** — SplitText infrastructure module.

IIFE pattern with 'use strict' (matching Phase 1 convention). Register via `window.RR.register('textSystem', {init, kill, refresh})`.

The module must:
1. **CDN graceful degradation**: Check `typeof SplitText !== 'undefined'` before any SplitText usage. If missing, expose a no-op fallback so hero.js doesn't break.
2. **Expose `window.RR.textSystem`** object with:
   - `split(element, options)` — Wraps `SplitText.create(element, options)` with `document.fonts.ready` gating via `window.RR.onFontsReady()`. Returns a Promise that resolves with the SplitText instance. Default options: `{ type: 'chars,words,lines', linesClass: 'rr-line', wordsClass: 'rr-word', charsClass: 'rr-char' }`. CRITICAL: set `mask: 'overflow'` in SplitText options to get masked overflow:hidden wrappers automatically (this is the GSAP SplitText built-in masking feature — do NOT manually add overflow:hidden CSS).
   - `killAll()` — Reverts all tracked SplitText instances (call `.revert()` on each).
   - `resplitAll()` — Kills then re-splits all tracked elements (for responsive resize).
3. **Auto-resplit on resize**: Use a debounced (250ms) resize handler that calls `resplitAll()` then `ScrollTrigger.refresh()`. This addresses TEXT-04.
4. **Track all splits** in an internal array so `killAll()` and `resplitAll()` can iterate.
5. **Reduced motion**: If `window.RR.state.hasReducedMotion` is true, the `init()` method should be a no-op (text remains static).

**Create `assets/animation/hero.js`** — Hero opening sequence master timeline module.

IIFE pattern with 'use strict'. Register via `window.RR.register('hero', {init, kill, refresh})`.

The module must:
1. **CDN graceful degradation**: Check `typeof gsap !== 'undefined'` and `typeof SplitText !== 'undefined'`. If GSAP missing, return. If SplitText missing, fall back to simple opacity/y tweens (no character split).
2. **Reduced motion gate** (HERO-05): If `window.RR.state.hasReducedMotion`, immediately `gsap.set()` all hero elements to visible state (`clearProps: 'all'`) and return — no animation.
3. **Homepage preloader handoff** (HERO-01, HERO-04): Detect homepage via `window.RR.page === 'home'`. On homepage:
   - Wait for the existing preloader exit to complete. The existing shared.js `exitLoader()` adds CSS class `exit-bg` to `#brandLoader` and then `dismissed` 900ms later. Hook into this by using a MutationObserver on `#brandLoader` watching for the `exit-bg` class addition (this is the moment the loader background starts fading). When `exit-bg` is detected, begin the hero master timeline immediately — overlapping with the loader fade creates a seamless handoff.
   - If `#brandLoader` is not found or already has `dismissed` class (e.g., page cached), start immediately.
4. **Hero master timeline** (HERO-02, HERO-03, HERO-04): Build a single `gsap.timeline()` that:
   - a) Use `window.RR.textSystem.split()` on the hero title element (`.hero-title` or `.about-hero-title` or `.sub-hero-title` depending on page). Await the Promise.
   - b) Animate SplitText chars with masked stagger: `gsap.fromTo(chars, {yPercent: 100}, {yPercent: 0, duration: 0.8, stagger: 0.02, ease: 'power3.out'})` — the mask: 'overflow' option on SplitText creates overflow:hidden wrappers, so yPercent:100 slides chars up from below the mask.
   - c) Fade in hero tag/subtitle/CTAs/scroll-hint with staggered y+opacity (similar to current but tighter timing).
   - d) Total timeline duration must be under 2.5 seconds from first visible movement.
5. **Non-homepage pages** (no preloader): Start hero timeline immediately on DOMContentLoaded via `window.RR.onFontsReady()`.
6. **Page-specific selectors**: Use a selector map based on `window.RR.page`:
   - `'home'`: title=`#heroTitle`, tag=`#heroTag`, sub=`#heroSub`, actions=`#heroActions`, scroll=`#heroScroll`
   - `'about'`: title=`#aboutTitle`, tag=`#aboutTag`, sub=`#aboutSub`, scroll=`#aboutScroll`
   - All capability pages + contact + careers: title=`#heroTitle`, tag=`#heroTag`, sub=`#heroSub`, scroll=`#heroScroll`
7. **kill()**: Kill the master timeline, revert any SplitText instances created by this module.
8. **refresh()**: Re-split text elements and rebuild timeline (called by matchMedia on breakpoint change).

IMPORTANT constraints:
- Do NOT modify shared.js — the existing preloader logic stays. hero.js hooks into its completion.
- Do NOT use nested ScrollTrigger inside the hero timeline (Pitfall 3 from research).
- Use `gsap.fromTo()` not `gsap.from()` to prevent FOUC (Pitfall 8).
- The hero title's FOUC prevention is already handled by controller.js (`gsap.set(heroTitle, {opacity:0, y:30})`). hero.js should animate FROM that hidden state TO visible. Use `gsap.fromTo` with `{opacity:0, y:0}` -> `{opacity:1, y:0}` for the overall element, but the character-level animation uses `yPercent` within SplitText masks.
- For the SplitText character reveal: first set the parent element opacity to 1 (so chars are visible within their masks), then animate individual chars with `yPercent: 100 -> 0`. The parent opacity:0 from FOUC prevention should be cleared to 1 at the start of the char animation.
