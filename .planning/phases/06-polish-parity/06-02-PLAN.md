# Plan 06-02: Horizontal Scroll Showcase & Final Integration Audit

## Objective

Create a horizontal scroll showcase section for the capabilities grid on index pages using GSAP ScrollTrigger pin-scrub. The section pins as the user scrolls and the content scrubs horizontally through all five capability cards. Then perform a final integration audit verifying all Phase 6 success criteria across the complete animation system.

## must_haves

- [ ] Capabilities section on index.html pins and scrubs horizontally through all 5 cap-cards
- [ ] Horizontal scroll works identically on `/cn/index.html` and `/id/index.html`
- [ ] Pinning does not cause layout shift or jank — smooth 60fps scrub
- [ ] On mobile (coarse pointer), horizontal scroll is disabled — cards display in normal vertical grid layout
- [ ] `will-change` is applied only during active scrub and removed when section unpins
- [ ] Lenis smooth scroll does not conflict with ScrollTrigger pin — `normalizeScroll` or Lenis pause handled correctly during pin
- [ ] All 5 Phase 6 success criteria from ROADMAP pass

## Tasks

<task id="1">
**Create horizontal-showcase.js module**

Create a new animation module at `assets/animation/horizontal-showcase.js` following the established RR module pattern (IIFE, strict mode, init/kill/refresh lifecycle, RR.register).

Implementation:

1. **Container setup**: Target `.caps-section` on index pages. The existing `.caps-grid` (5 `.cap-card` elements) becomes the horizontal scroll track.

2. **CSS modifications in shared.css**:
   - Add `.caps-section.is-horizontal` class that activates horizontal layout:
     ```css
     .caps-section.is-horizontal .caps-grid {
       display: flex;
       flex-wrap: nowrap;
       gap: 24px;
       width: max-content;
     }
     .caps-section.is-horizontal .cap-card {
       flex: 0 0 clamp(280px, 30vw, 400px);
     }
     ```
   - Keep the existing grid layout as the default (mobile fallback)

3. **GSAP ScrollTrigger pin-scrub**:
   ```javascript
   // Calculate horizontal distance: total width - viewport width
   var scrollDistance = grid.scrollWidth - grid.parentElement.offsetWidth;

   gsap.to(grid, {
     x: -scrollDistance,
     ease: 'none',
     scrollTrigger: {
       trigger: capsSection,
       start: 'top top',
       end: '+=' + scrollDistance,
       pin: true,
       scrub: 0.5,
       anticipatePin: 1,
       invalidateOnRefresh: true,
       onToggle: function(self) {
         // Manage will-change lifecycle (PERF-03)
         grid.style.willChange = self.isActive ? 'transform' : 'auto';
       }
     }
   });
   ```

4. **Lenis integration**: When the section pins, Lenis scroll events need to feed into ScrollTrigger correctly. The existing scroll-engine.js already syncs Lenis with ScrollTrigger ticker, so no additional work is needed. However, if edge cases arise:
   - Use `ScrollTrigger.normalizeScroll(true)` only if Lenis pin interaction has issues
   - The STATE.md concern about Lenis + ScrollTrigger + horizontal pin is addressed by testing

5. **Mobile gate**: Exit init() early if `window.RR.breakpoints.isCoarsePointer` is true — mobile keeps vertical grid layout.

6. **Reduced motion gate**: If reduced motion, skip pin-scrub entirely — cards display in normal grid.

**Verification:**
- Scroll past capabilities section on desktop: section pins, cards scroll horizontally
- All 5 cards are accessible during horizontal scrub
- Unpinning restores normal scroll flow
- Mobile: vertical grid layout, no pin behavior
</task>
